🔗 2. HOW EVERYTHING CONNECTS (PK / FK EXPLAINED)
🧠 Core Concept

Primary Key (PK) = unique identifier of a table
Foreign Key (FK) = reference to another table

🔄 Relationship Flow (Important)
👥 Users ↔ NGOs
users.ngo_id → ngos.id
A user belongs to an NGO
NGO has many users

👉 One-to-Many

📍 Locations ↔ Everything
users.location_id → locations.id
problems.location_id → locations.id
resources.location_id → locations.id

👉 One location reused everywhere

🚨 Problems ↔ Users
problems.reported_by → users.id

👉 One user can report many problems

📋 Tasks ↔ Problems
tasks.problem_id → problems.id

👉 One problem → multiple tasks

🤝 Task Assignments
task_assignments.task_id → tasks.id
task_assignments.user_id → users.id

👉 Many users ↔ many tasks (Many-to-Many bridge table)

🛠 Skills ↔ Users
user_skills.user_id → users.id
user_skills.skill_id → skills.id

👉 Many-to-Many relationship

📦 Resources ↔ Tasks
resource_allocations.resource_id → resources.id
resource_allocations.task_id → tasks.id
🤖 AI Matches
ai_matches.problem_id → problems.id
ai_matches.user_id → users.id

👉 AI links problems to users

🔔 Notifications
notifications.user_id → users.id

👉 Each notification belongs to a user
